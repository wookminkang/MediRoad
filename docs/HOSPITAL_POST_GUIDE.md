# 병원 공식 안내형 포스팅 운영 가이드

> 라우트: `/hospitals/[slug]/posts/[postId]` (실제 구현 기준)
> 데이터: `hospital_posts` 테이블 (시드/운영자 입력, service_role)
> 상위 설계: [HOSPITAL_CONTENT_SEO.md](./HOSPITAL_CONTENT_SEO.md)

## 목적 — "우리 병원 이용 설명서"

병원이 직접 작성하는 **공식 안내 콘텐츠**. 방문 예정자가 **언제·어디로·어떻게 이용할지** 판단하도록 돕는다.
질환·증상 일반 정보를 길게 설명하지 않고, **해당 병원에만 있는 이용·운영 정보**에 집중한다.

## 콘텐츠 역할 구분 (중복 금지)

| 구분 | 역할 | 대표 주제 |
|---|---|---|
| 병원 상세 페이지 | 사실 정보 기준 | 진료시간·주소·진료과목·지도·전화 |
| **병원 공식 포스팅** | **방문·진료·시설·운영 안내** | 초진 안내, 예약/접수, 주차, 대중교통, 의료진 일정, 공지 |
| 건강 칼럼(columns) | 공통 건강 정보 | 증상·질환·예방·생활습관 |

> ⚠️ "감기 증상", "고혈압 관리법" 같은 **질환 일반정보는 건강 칼럼**에 작성. 병원 포스팅은 "우리 병원에서 그 진료를 받으려면 무엇을 확인해야 하는지"에 집중.

## 작성 원칙

1. **병원에만 있는 정보 우선** — 접수 마감, 예약/당일접수 방식, 주차 위치·비용, 역 출구·도보 동선, 검사 전 준비, 의료진 일정
2. **이용자 질문에 먼저 답** — "예약 필요?", "토요일 진료?", "어느 역에서 몇 분?", "초진 준비물?", "주차 가능?"
3. **확인 가능한 사실만** — 효과·완치·최고·비교우위 단정 금지, 운영정보는 최종 확인일 명시
4. **건강 칼럼과 중복 금지**

## 추천 카테고리

- **A. 방문·이용 안내**: 예약·접수·주차, [역]→병원 동선, 접수마감/당일진료, 보호자·유아차·휠체어
- **B. 진료·검사 과정**: 초진 진행순서, 검진 예약~결과, 검사 전 금식·약 확인, 서류 발급
- **C. 의료진·시설·운영**: 의료진 일정, 대기/진료/검사실 이용, 감염관리, 신규 장비
- **D. 지역 밀착**: [지역] 직장인 야간진료, [역] 인근 방문 전 확인, 공휴일 진료 확인
- **E. 공식 공지**: 휴진·진료시간 변경, 이전·주차 변경, 예약정책 변경 (단기 공지는 종료 후 분리)

## 본문 구조 (답변 우선 — AI 인용 최적화)

1. **요약(summary, 2~5줄)** = 한눈에 보기(위치/가까운 역/예약/접수마감/주차)
2. 질문형 H2/H3 → **한 줄 직답** → 상세
3. 병원명·지역·역 엔티티를 방문 맥락에서 자연스럽게 명시
4. 하단 **FAQ 3~5개** + 병원 상세/진료시간/오시는길 **내부링크**
5. 작성 주체·최종 확인일·면책 문구

## 제목 템플릿

| 유형 | 템플릿 |
|---|---|
| 처음 방문 | `{병원명} 처음 방문 전 예약·접수·주차 안내` |
| 오시는 길 | `{역명}에서 {병원명}까지 오는 방법｜대중교통·주차` |
| 진료시간 | `{병원명} 평일·토요일 진료시간과 접수 마감 안내` |
| 검사 준비 | `{검사명} 전 준비 사항｜{병원명} 검사 안내` |
| 의료진 | `{병원명} {진료과목} 의료진 진료 일정 안내` |
| 지역 안내 | `{지역명}에서 {진료과목} 진료를 계획한다면 확인할 정보` |
| 공지 | `{병원명} {월}월 진료시간 및 휴진 안내` |

## DB 필드 매핑 (스키마 변경 없음 — 시드 객체 기준)

가이드 템플릿(frontmatter)의 개념 필드를 **실제 `hospital_posts` 컬럼**에 매핑한다.

| 가이드 필드 | hospital_posts 컬럼 | 비고 |
|---|---|---|
| title | `title` | |
| slug/postSlug | `id` | URL의 postId (영문 케밥, 병원 내 고유) |
| hospitalSlug | (저장 안 함) | `hospital_id`로 연결, 라우트는 병원 slug 사용 |
| category | `tags[0]` | 별도 컬럼 없음 → 첫 태그로 표현(예: "방문안내") |
| tags | `tags[]` | `{지역} {진료과}`, 병원명, 안내 유형 |
| description | `excerpt` | meta_description 폴백 |
| 한눈에 보기 | `summary[]` | TL;DR 불릿(GEO 스니펫) |
| author{name} | `author{ name, role }` | role=병원 유형(예: "한방병원") |
| reviewer | `reviewed_by{ name, specialty }` | role→specialty |
| coverImage | `thumbnail` | 실제 병원 공간 사진 권장 |
| publishedAt | `published_at` | |
| updatedAt / sourceUpdatedAt | `updated_at` | 트리거 자동 + 본문 "최종 확인일" 표기 |
| relatedLinks | 본문 내부링크 + `refs[]` | 별도 컬럼 없음 → 본문 하단 링크로 작성 |
| isNotice | `tags`에 "공지" 포함 | 별도 컬럼 없음 |
| canonical | (앱 자동) | self canonical 자동 생성 |
| noindex | `noindex` | draft/hidden은 자동 noindex |
| status | `status` | published/draft/hidden |
| conditions/related_departments | 보통 비움 | 안내형은 질환 결속 약함 |

> 입력은 시드 스크립트(`scripts/seed-*.mts`)로 `hospital_posts`에 upsert. (마크다운 ingest 파이프라인은 칼럼 전용)

## 표현 가이드

**권장**: "방문 전 확인해 주세요", "운영 상황에 따라 달라질 수 있습니다", "전화/예약 채널에서 확인할 수 있습니다", "이 글은 병원 이용 정보를 정리한 안내입니다"

**금지**: "가장 잘하는 병원", "확실한 치료 효과", "완치 가능", "타 병원보다 우수", "무조건 추천", "부작용 없음"

## 의료광고법 체크 (게시 전)

- [ ] 효과 보장·단정 표현 없음
- [ ] 환자 치료 경험담/후기 없음
- [ ] 비교·최상급("최고/유일") 없음
- [ ] 비급여 할인·이벤트 유인 없음
- [ ] 의학적 면책(개인차·전문의 상담) 명시
- [ ] 주소·전화·진료시간이 상세 페이지와 일치(최신)
- [ ] 제목·H1이 다른 글과 중복 안 됨
